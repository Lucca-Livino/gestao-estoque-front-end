# Plano de Teste - Sistema de Gestão de Estoque

## 1. Introdução

**Objetivo**: Garantir a qualidade e funcionalidade do sistema de gestão de estoque, cobrindo frontend (Next.js), backend (Node.js) e integrações.

**Escopo**: Testes E2E com Cypress para funcionalidades principais: autenticação, fornecedores, produtos, movimentações e dashboard.

**Premissas**: Ambiente Docker, dados de seed, usuário admin logado.

## 2. Módulos a Serem Testados

- Autenticação
- Fornecedores
- Produtos
- Movimentações
- Dashboard

## 3. Abordagem

- E2E com Cypress.
- Testes positivos e negativos.
- Relatórios automáticos.

## 4. Cronograma

- Mês 1: Autenticação e fornecedores.
- Mês 2: Produtos e movimentações.
- Mês 3: Dashboard e integração.

## 5. Riscos

- Dependências de API.
